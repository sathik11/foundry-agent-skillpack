## What runs in your tenant

| Layer | Provider |
| --- | --- |
| Image build, agent create, version create | `azd ai agent` extension (you run it) |
| Per-capability preflight, RBAC dispatch, drift detection | This skillpack's prompts (your coding agent runs them) |
| Foundry-native eval rules, scheduled eval, cloud red-team | Convergent lifecycle scripts in this skillpack (`ensure_*` pattern) |
| Runtime guardrails (middleware, Content Safety, Purview DLP) | Vendored Python middleware — copied into your agent's container |
| Knowledge sources, MCP servers, network class | Your existing Azure resources; skillpack verifies and grants RBAC |

## What sets this apart

`azd ai agent` deploys hosted agents. Foundry runs them. **This skillpack is everything in between** — the lifecycle, the governance, the audit trail, the brownfield path. Six things you don't get anywhere else today:

## Install

```bash
# 0. (One-time) Initialize apm.yml so APM knows where to deploy skills + prompts.
#    Idempotent — only writes if apm.yml is missing.
[ -f apm.yml ] || cat > apm.yml <<'EOF'
name: my-foundry-project
version: 0.0.1
targets: [copilot, agent-skills]   # add claude / cursor / windsurf as needed
EOF

# 1. Install the engineering skillpack — knowledge + prompts + scripts
apm install sathik11/Foundry-Hosted-Agent-Skill/foundry-agent-skillpack

# 2. Optional: install the fixtures package — recipes + runnable fixtures
apm install sathik11/Foundry-Hosted-Agent-Skill/foundry-agent-fixtures
```

> **Already have `apm.yml`?** Make sure it has a `targets:` line (e.g. `targets: [copilot, agent-skills]`) — APM no longer auto-defaults to `copilot` and will refuse to install otherwise. Full target list: see [Install → Choosing targets](/getting-started/install/#troubleshooting--no-harness-detected).

Then open a coding agent that reads APM-installed prompts (Copilot Chat with `.github/prompts/`, Claude Code with `.claude/commands/`, Cursor with `.cursor/skills/`, Windsurf with `.windsurf/skills/`, OpenCode, Gemini, or Codex via `AGENTS.md`) and run:

```text
/plan-agent name=hello-foundry description="..."
/prepare-deploy agent_path=agents/hello-foundry
azd up
/configure-rbac agent_path=agents/hello-foundry agent_name=hello-foundry
/verify-agent agent_name=hello-foundry test_query="Hi" agent_path=agents/hello-foundry
/setup-evals agent_name=hello-foundry agent_path=agents/hello-foundry
```

The first 30 minutes are documented step-by-step in the [Greenfield Quickstart](/getting-started/greenfield/) (or read the underlying recipe [on GitHub](https://github.com/sathik11/Foundry-Hosted-Agent-Skill/blob/main/foundry-agent-fixtures/.apm/skills/foundry-agent-fixtures/recipes/01-greenfield-quickstart.md)).

## What this is not

- Not a runtime. The skillpack ships *knowledge*, *prompts*, and *vendored middleware code*. The agent itself runs in your Foundry container.
- Not a replacement for `azd ai agent`. The skillpack validates and dispatches; `azd up` deploys.
- Not a UI. The deliverables are markdown skills, slash commands, and per-agent state files. You read the reports; the skillpack doesn't render dashboards.
- Not Microsoft-published. This is community work tracking the Foundry hosted agent surface — see the [roadmap](/roadmap/) for the path to Microsoft Learn submission.
